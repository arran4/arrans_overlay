Please update the `overlay_workflow_builder` tool to address the following issues observed when generating Github Binary Release workflows for Gentoo overlays:

1. **Tag Prefix Handling**: Currently, the generated workflow assumes all GitHub release tags are prefixed with a 'v' (e.g., `v1.0.0`). The bash logic `version="${tag#v}"; if [ "${version}" = "${tag}" ]; then ... continue; fi` incorrectly skips tags that do not use the 'v' prefix (like `1.0.0`). Furthermore, the generated `SRC_URI` in the ebuild template hardcodes `v${PV}`. The generator should be updated to intelligently handle or allow configuration for whether to expect a 'v' prefix, to support repositories like `google-antigravity/antigravity-cli` that use plain version tags.
2. **Metadata XML YAML Formatting**: The generated `metadata.xml` heredoc block in the workflow file outputs invalid YAML syntax because the XML tags (like `<maintainer>`, `<use>`, etc.) lack the required indentation to align with the YAML block structure. This causes `actionlint` and YAML parsers to fail. Please ensure the generated XML heredoc is fully indented (e.g., at least 10 spaces).
3. **Missing Maintainer Block**: The generated `metadata.xml` often lacks a `<maintainer>` block entirely, which causes Gentoo CI `pkgcheck` to fail with "Package has no maintainers defined". Ensure a default maintainer block (e.g., `gentoo@arran4.com`) is correctly formatted and included in the `<pkgmetadata>` block.
4. **Checkout Action Version**: The tool generates `uses: actions/checkout@v2`, which runs on deprecated Node.js versions and causes CI warnings. It should be updated to use `uses: actions/checkout@v4`.
